DMA with SPI driver FLASH(W25Q128)

2022-03-22

REVIEW AND SUMMARY:

W25Q128 just need to look at the instructions in document in detail. so I review the SPI and DMA in detail

SPI review:

SPI data transmit block diagram:

follow the above picture: SPI master and slave both have one shift register, in duplex mode, peripherals are write and read synchronous . if only write, the master ignore receive data. if only read master must send the NULL byte to synchronous the receive byte.

SPI phase:

SPI CR (control register) have two bit to control the SPI phase: CPOL and CPHA

  • CPOL control the SPI clock free level, if CPOL == 0,SPI clock free level is low level. On the contrary, CPOL == 1, SPI clock free level is high level.
  • CPHA control the SPI sampling timing. if CPHA == 0, SPI sample at falling edge. On the contrary, CPHA == 1, SPI sample at the rising edge.

SPI block diagram:

SPI read only mode:

when the SPI master device want to read data form SPI Slave, we have to send the 0xFF byte to synchronous the receive data, it will occupying the master processor resource. we can she the SPI_CR2 register`s SPE bit to set the SPI to only read mode, it will activate the SCK clock and receive data serially. we just only need to receive data and we do not need to send the 0xFF byte to synchronous.

SPI DMA mode:

when we use SPI send/receive bulk data, it will occupy a lot of CPU resources. we can enable the SPI DMA request bit to use DMA to control SPI.

  • In the process of sending, each time when the SPI TXE change to 1 by the hardware, it will send the write request to the DMA and DMA will execute write option.
  • In the process of reading, each time when the SPI RXNE change to 1 by the hardware, it will send the write request to the DMA and DMA will execute read option.

DMA review:

DMA block diagram:

  • in stm32f407, DMA1 have two AHB bus interface to connect memory and peripherals.(上图的存储器接口和外设接口). Usually, the peripherals interface connect to the APB bus so we change use like SPI, IIC, UART protocol to control the peripherals.
  • DMA has FIFO mode and direct mode:
    • FIFO mode: we should specify the FIFO threshold value like 1/4, 2/4, 3/4, full FIFO
      • peripherals to memory: every time when peripherals request start-up, peripherals will send data to the FIFO. when the FIFO threshold reach, FIFO will send the data of the FIFO to the memory.
      • memory to peripherals: when the time enable the memory to peripherals used FIFO, memory will send data to FIFO until the FIFO full, each time the peripherals request come, the FIFO will remove the data to the target peripherals
    • direct mode:
      • peripherals to memory: each time finish the data transmit form peripherals, the data will removed form FIFO to the memory.
      • memory to peripherals: in this mode, we should specify the MSIZE, when the time enable the memory to peripherals directly, memory will send data to FIFO until reach the MSIZE. after the peripherals request data, the FIFO will be overload again.

3.USART note

2022-03-08

  • SR : status register

    • [0] PE: when parity err occur in the receive mode, this bit will set 1 by hardware
    • [1] FE: when framing err occur, this bit will set 1 by hardware
    • [2] NF: when detected noise, this bit wil set 1 by hardware
    • [3] ORE: overrun err occur, set 1 by hardware
    • [4] IDLE: when the idle line was detected, set 1 by hardware
    • [5] RXNE: read data register not empty, when data in the RDR register was transmitted to DR register, this register was set 1, at this time, if the IDLEIE bit in the CR1 register has been set 1, it will produce interrupt.
    • [6] TC: transmission complete, set 1 by hardware
    • [7] TXE: transmit data register empty, set 1 by hardware
    • [8] LBD: LIN break detecting flag
    • [9] CTS: CTS flag

  • DR : data register

  • BRR : baud data resister

    • [11:0]DIV_Mantissa, USARTDIV`s mantissa
    • [3:0]DIV_Fraction, USARTDIV`s fraction

  • CR1: control register1

  • CR2: control register2

  • CR3: control register3

  • CTRP: usart guard time and prescaler register

  • USART_BaudRate: this member configures the usart communication baud rate.
    the baud rate computed using the following formula:
    - inter divider = (PCLKZx) / ( 8 * (OVR8 + 1) * (USART_InitStrut->USART_BaudRate) )
    - fractional divider = ((intergerdivider - ((u32)integerdivider)) * 8 * (OVR8 + 1)) + 0.5
    - where OVR8 is the “oversampling by 8 mode” configuration bit in the CR1 register
  • USART_Wordlength: specifies the number of data bits transmitted or received in a frame.
  • USART_StopBIts: specifies the number of stop bits transmitted.
  • USART_Parity: specifies the paritymode.
  • USART_Mode: specifies whether the receive or transmit mode is enabled or disabled.
  • USART_HardwareflowControl: specifies whether the hard flow control mode is endable or disable.
  • USART_Clock: specifies whether the usart clock is enable or disable.
  • USART_CPOL: specifies the steady state of the serial clock
  • USART_CPHA: Specifies the clock transition on which the bit capture is made.
  • USART_LastBit: Specifies whether the clock pulse corresponding to the last transmitted data bit (MSB) has to be output on the SCLK pin in synchronous mode.

Introduction

  • data bit length: 8-9bit
  • support 1-2 stop-bit
  • support DMA buffer transfer
  • interrupt support:
    • transmission interrupt
      • buffer full interrupt
      • buffer empty interrupt
      • transmission finish interrupt
    • four err flag check interrupt
      • overflow flag
      • noise flag
      • frame err flag
      • parity err flag

Communication process

UART PARAM

  • start bit
  • data length [8-9(if has parity bit)]
  • parity bit (the ninth bit)
  • stop bit length
  • baud

Important register

  • SR register
  • recive register
  • send register
  • BRR: baud register
  • baud = f(pclkx)/(16*USART_DIV)

Hardware flow control

serial communication protocol is a kind of asynchronous communication protocol, so its data may be wrong due with the clock error. hardware flow control attend to deal with this problem.

hardware flow control need extra two pins to control the communication flow.

Take receive data for example:

when the serial communication is not finished, the RTS pin is pull up to tell the sender wait until finish, after receive completed, the RTS pin is pull down to tell the sender this communication is finish and this is prepare to receive next data.

2.DMA note

2022-03-07

DMA note base stm32f407 manual

Introduction:

  • DMA provide high-speed data transmission between peripheral and mem and between mem and mem transport.
  • DMA can transmit data quickly without any cpu operation, which save resources from cpu.
  • DMA controller combines the powerful dual AHB main bus architecture with independent FIFO together, the system bandwidth is optimized
  • the stm32`s two DMA controllers have a total of 16 data stream (8 stream for each DMA controller), each DMA controller used to manager one or many mem access request. each data stream has a total of 8 channel (request). each channel has one arbiter, which used to handle the priority of the DMA request.

Features:

  • dual AHB main bus architecture, one for mem access and the other for peripheral access.
  • AHB slave programming interface that only supports 32-bit access
  • each DMA controller has eight data stream, each data stream has up to eight (or requests)
  • each data stream has a separate 4 level 32-bit FIFO mem buffer, which can be used in FIFO mode or direct connection mode.
    • FIFO mode: the threshold level can be selected as 1/4, 1/2 or 3/4 of the FIFO size by software.
    • direct mode: each DMA request immediately initiates a transfer to memory. when the DMA request is configured to transmit data in memory to peripheral mode in direct mode (FIFO prohibited), DMA will only pre load one data from memory it is loaded into the internal FIFO to ensure that the data is transmitted immediately once the peripheral trigger the DMA request.
  • each data stream can be configured by hardware to
    • support the common channel between peripheral and mem and between mem and mem.
    • it also support double buffer channels with double buffers on the memory side.
  • each data stream also supports mem to mem transmission triggered by software (DMA2 controller only).
  • 5 event flag (DMA半传输， DMA传输完成、DMA传输错误、DMA FIFO错误、 直接模式错误)

  • 5.低功耗管理复位

基于 TIM5/TIM11 的内部/外部时钟测量

  • TIM5

    基于 TIM5 channel4 的内部/外部时钟测量
    TIM5 具有一个输入复用器，可选择输入捕获是由 I/O 触发还是由内部时钟触发。此选择通过
    TIM5_OR 寄存器的 TI4_RMP [1:0] 位执行。
    将 LSE 连接到 channel4 输入捕获的主要目的是为了精确测量 HSI（这需要将 HSI 用作系统
    时钟源）。借助 LSE 信号连续边沿之间的 HSI 时钟计数数量，即可对内部时钟周期进行测
    量。利用 LSE 的高精度（通常为几十 ppm），用户能以同一分辨率测定时钟频率，并可通
    过对时钟源进行微调来补偿由生产工艺造成的和/或与温度和电压相关的频率偏差。
    HSI 振荡器设有针对此目的的专用校准位，且支持用户访问。
    其基本原理是基于相对的测量（例如， HSI/LSE 比）：因此，精度与两个时钟源之比紧密相
    关。比率越大，测量效果越好。
    同时也可测量 LSI 频率：这对于没有晶振的应用场合非常实用。超低功耗 LSI 振荡器具有较
    大的生产工艺偏差：通过测量该振荡器与 HSI 时钟源的比率，可以借助 HSI 精度为其测定频
    率。可通过此测量值实现更加精确的 RTC 时基超时（当使用 LSI 作为 RTC 时钟源时）和/或
    实现精度水平可以接受的 IWDG 超时。
    LSI 频率通过以下步骤测量：

    1. 使能 TIM5 定时器并将 channel4 配置为输入捕获模式。
    2. 将 TIM5_OR 寄存器中的 TI4_RMP 位设置为 0x01，以在内部将 LSI 时钟连接到 TIM5
      channel4 输入捕获来实现校准。
    3. 通过 TIM5 捕获/比较 4 事件或中断测量 LSI 时钟频率。
    4. 使用测得的 LSI 频率来按照所需的时基更新 RTC 的预分频器，并且/或者用其来计算
      IWDG 超时。

  • TIM11

    TIM11 具有一个输入复用器，可选择输入捕获是由 I/O 触发还是由内部时钟触发。此选择通
    过 TIM11_OR 寄存器的 TI1_RMP [1:0] 位执行。 HSE_RTC 时钟（由一个可编程预分频器分
    频的 HSE）连接到通道 1 输入捕获，以粗略指示外部晶振频率。这要求 HSI 为系统时钟
    源。此功能非常实用，例如可借此测定谐波频率或分谐波频率（ —50/+100% 偏差），进而确
    保符合 IEC 60730/IEC 61335 标准 。

RCC寄存器

uint32_t

  • CR

    • RCC clock control register

      • PLLI2SRDY

        位 27 PLLI2SRDY： PLLI2S 时钟就绪标志 (PLLI2S clock ready flag)
        由硬件置 1，用以指示 PLLI2S 已锁定。
        0： PLLI2S 未锁定
        1： PLLI2S 已锁定

      • PLLI2SON

        位 26 PLLI2SON： PLLI2S 使能 (PLLI2S enable)
        由软件置 1 和清零，用于使能 PLLI2S。
        当进入停机或待机模式时由硬件清零。
        0： PLLI2S 关闭
        1： PLLI2S 开启

      • PLLRDY

        位 25 PLLRDY：主 PLL (PLL) 时钟就绪标志 (Main PLL (PLL) clock ready flag)
        由硬件置 1，用以指示 PLL 已锁定。
        0： PLL 未锁定
        1： PLL 已锁定

      • PLLON

        位 24 PLLON：主 PLL (PLL) 使能 (Main PLL (PLL) enable)
        由软件置 1 和清零，用于使能 PLL。
        当进入停机或待机模式时由硬件清零。如果 PLL 时钟用作系统时钟，则此位不可清零。
        0： PLL 关闭
        1： PLL 开启

      • CSSON

        位 19 CSSON：时钟安全系统使能 (Clock security system enable)
        由软件置 1 和清零，用于使能时钟安全系统。当 CSSON 置位时，时钟监测器将在 HSE 振
        荡器就绪时由硬件使能，并在检出振荡器故障时由硬件禁止。
        0：时钟安全系统关闭（时钟监测器关闭）
        1：时钟安全系统打开（如果 HSE 振荡器稳定，则时钟监测器打开；如果不稳定，则关闭）

      • HSEBYP

        位 18 HSEBYP： HSE 时钟旁路 (HSE clock bypass)
        由软件置 1 和清零，用于用外部时钟旁路振荡器。外部时钟必须通过 HSEON 位使能才能为
        器件使用。
        HSEBYP 只有在 HSE 振荡器已禁止的情况下才可写入。
        0：不旁路 HSE 振荡器
        1：外部时钟旁路 HSE 振荡器

      • HSERDY

        位 17 HSERDY： HSE 时钟就绪标志 (HSE clock ready flag)
        由硬件置 1，用以指示 HSE 振荡器已稳定。在将 HSEON 位清零后， HSERDY 将在 6 个
        HSE 振荡器时钟周期后转为低电平。
        0： HSE 振荡器未就绪
        1： HSE 振荡器已就绪

      • HSEON

        位 16 HSEON： HSE 时钟使能 (HSE clock enable)
        由软件置 1 和清零。
        由硬件清零，用于在进入停机或待机模式时停止 HSE 振荡器。如果 HSE 振荡器直接或间接
        用于作为系统时钟，则此位不可复位。
        0： HSE 振荡器关闭
        1： HSE 振荡器打开

      • HSICAL

        位 15:8 HSICAL[7:0]：内部高速时钟校准 (Internal high-speed clock calibration)
        这些位在启动时自动初始化。

      • HSITRIM

        位 7:3 HSITRIM[4:0]：内部高速时钟微调 (Internal high-speed clock trimming)
        通过这些位，可在 HSICAL[7:0] 位基础上实现可由用户编程的微调值。可通过编程使其适应
        电压和温度的差异，使内部 HSI RC 的频率更为准确。

      • HSIRDY

        位 1 HSIRDY：内部高速时钟就绪标志 (Internal high-speed clock ready flag)
        由硬件置 1，用以指示 HSI 振荡器已稳定。在将 HSION 位清零后， HSIRDY 将在 6 个 HSI
        时钟周期后转为低电平。
        0： HSI 振荡器未就绪
        1： HSI 振荡器已就绪

      • HSION

        位 0 HSION：内部高速时钟使能 (Internal high-speed clock enable)
        由软件置 1 和清零。
        由硬件置 1，用于在脱离停机或待机模式时或者在直接或间接用作系统时钟的 HSE 振荡器发
        生故障时强制 HSI 振荡器打开。如果 HSI 直接或间接用于作为系统时钟，则此位不可清零。
        0： HSI 振荡器关闭
        1： HSI 振荡器打开

  • PLLCFGR

    • RCC PLL configuration register

      此寄存器用于根据公式配置 PLL 时钟输出：
      ● f(VCO 时钟) = f(PLL 时钟输入) × (PLLN / PLLM)
      ● f(PLL 常规时钟输出) = f(VCO 时钟) / PLLP
      ● f(USB OTG FS, SDIO, RNG 时钟输出) = f(VCO 时钟) / PLLQ

      • PLLQ

        位 27:24 PLLQ： 主 PLL (PLL) 分频系数，适用于 USB OTG FS、 SDIO 和随机数发生器时钟 (Main
        PLL (PLL) division factor for USB OTG FS, SDIO and random number generator clocks)
        由软件置 1 或清零，用于控制 USB OTG FS 时钟、随机数发生器时钟和 SDIO 时钟的频
        率。这些位应仅在 PLL 已禁止时写入。
        小心： | 为使 USB OTG FS 能够正常工作，需要 48 MHz 的时钟。对于 SDIO 和随即数生成
        器，频率需要低于或等于 48 MHz 才可正常工作。USB OTG FS 时钟频率 = VCO 频率 / PLLQ，并且 2 PLLQ 15
        0000： PLLQ = 0，错误配置
        0001： PLLQ = 1，错误配置
        0010： PLLQ = 2
        0011： PLLQ = 3
        0100： PLLQ = 4
        …
        1111： PLLQ = 15

      • PLLSRC

        位 22 PLLSRC： 主 PLL(PLL) 和音频 PLL (PLLI2S) 输入时钟源 (Main PLL(PLL) and audio PLL
        (PLLI2S) entry clock source)
        由软件置 1 和清零，用于选择 PLL 和 PLLI2S 时钟源。此位只有在 PLL 和 PLLI2S 已禁止时
        才可写入。
        0：选择 HSI 时钟作为 PLL 和 PLLI2S 时钟输入
        1：选择 HSE 振荡器时钟作为 PLL 和 PLLI2S 时钟输入

      • PLLP

        位 17:16 PLLP： 适用于主系统时钟的主 PLL (PLL) 分频系数 (Main PLL (PLL) division factor for main
        system clock)
        由软件置 1 和清零，用于控制常规 PLL 输出时钟的频率。这些位只能在 PLL 已禁止时写入。
        小心： 软件必须正确设置这些位，使其在此域中不超过 168 MHz。
        PLL 输出时钟频率 = VCO 频率 / PLLP 并且 PLLP = 2、 4、 6 或 8
        00： PLLP = 2
        01： PLLP = 4
        10： PLLP = 6
        11： PLLP = 8

      • PLLN

        位 14:6 PLLN： 适用于 VCO 的主 PLL (PLL) 倍频系数 (Main PLL (PLL) multiplication factor for VCO)
        由软件置 1 和清零，用于控制 VCO 的倍频系数。这些位只能在 PLL 已禁止时写入。写入这
        些位时只允许使用半字和字访问。
        小心： 软件必须正确设置这些位，确保 VCO 输出频率介于 192 和 432 MHz 之间。
        VCO 输出频率 = VCO 输入频率 × PLLN 并且 192  PLLN  432
        000000000： PLLN = 0，错误配置
        000000001： PLLN = 1，错误配置
        …
        011000000： PLLN = 192
        …
        110110000： PLLN = 432
        110110001： PLLN = 433，错误配置
        …
        111111111： PLLN = 511，错误配置

      • PLLM

        位 5:0 PLLM： 主 PLL (PLL) 和音频 PLL (PLLI2S) 输入时钟的分频系数 (Division factor for the main
        PLL (PLL) and audio PLL (PLLI2S) input clock)
        由软件置 1 和清零，用于在 VCO 之前对 PLL 和 PLLI2S 输入时钟进行分频。这些位只有在
        PLL 和 PLLI2S 已禁止时才可写入。
        小心： | 软件必须正确设置这些位，确保 VCO 输入频率介于 1 和 2 MHz 之间。建议选择
        2 MHz 的频率，以便限制 PLL 抖动。VCO 输入频率 = PLL 输入时钟频率 / PLLM 并且 2  PLLM  63
        000000： PLLM = 0，错误配置
        000001： PLLM = 1，错误配置
        000010： PLLM = 2
        000011： PLLM = 3
        000100： PLLM = 4
        …
        111110： PLLM = 62
        111111： PLLM = 63
